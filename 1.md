Image PDM - 图纸数据管理系统

1. 项目背景与客户需求
   背景： 制造业企业需要一个专业的产品数据管理（PDM）平台来管理海量的工程图纸和技术文档。

客户核心需求：

支持多种格式文件（PDF、TXT等）的统一管理和快速检索
实现智能化的图纸分类和标签管理（BOM管理、避免文件冗余）
提供高效的文档搜索能力，支持多维度查询
建立设备型号与关键词的智能关联体系
需要一个现代化、易用的用户界面2. 我在项目中的主要工作
技术架构设计与实现：

前端：Next.js + React + TypeScript + Tailwind CSS + Radix UI
后端：FastAPI + Uvicorn + Supabase (PostgreSQL)
存储：MinIO 对象存储 + 向量数据库
核心功能开发：

功能模块 技术实现 关键点
RAG 智能问答系统 向量化检索 + 语义相似度匹配 构建设备数据与文档知识库的向量索引；使用 pgvector 存储向量；通过语义相似度检索 top-k 相关文段；基于知识库精准回答
双策略设备检索 精确匹配 + 模糊匹配 设计型号优先级；关键词分词（jieba）；停用词过滤；在设备数据中做关键词匹配；提高召回率
组件化 UI 架构 shadcn/ui + Radix UI 封装 FileSelector、SaveFileDialog、FileList、BomManagement 等可复用组件；使用 react-resizable-panels 实现可拖拽布局；支持 BOM 面板与 Chat 区域灵活切换
前后端分离架构 Next.js App Router + FastAPI 采用 Next.js API Routes 与 FastAPI 双数据源；支持 RESTful API；配置 CORS；实现跨域访问与前后端解耦
JWT 用户认证 Token 签发与验证 基于 JWT 实现用户认证；登录成功后签发 Access Token；前端在请求头中携带 Token；后端通过中间件验证 Token 有效性；支持 Token 过期与刷新机制
技术亮点：

集成通义千问 API 和自定义 LLM，支持多模型切换
使用 PaddleOCR 进行 PDF 文本提取和识别
MinIO 文件存储管理，支持分布式部署
向量化搜索提升检索精度和用户体验 3. 项目成果
